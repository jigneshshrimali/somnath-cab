# Somnath Cab — Requirements

This project is a **Node.js / Next.js** application, not Python — so there is no `pip`-style
`requirements.txt` in the traditional sense. This file lists everything required to build and run
it, in the same spirit: system prerequisites first, then the exact package versions pinned in
`package.json` / `package-lock.json`.

Actual dependency resolution always happens via `package.json` — this file is a human-readable
reference so you can see what's required at a glance before running `npm install`.

---

## 1. System Requirements

| Requirement | Minimum Version | Notes |
|---|---|---|
| Node.js | 18.18.0 or later (20.x LTS recommended) | https://nodejs.org — includes npm |
| npm | 9.x or later | Bundled with Node.js |
| Git | any recent version | For cloning/version control |
| OS | Windows 10/11, macOS, or Linux | Guide below targets Windows + PowerShell |
| VS Code | latest | https://code.visualstudio.com |
| RAM | 4 GB minimum, 8 GB+ recommended | Next.js dev server + TypeScript checking |
| Disk | ~500 MB free | `node_modules` + build cache |

Optional (only needed once you wire up live integrations):
- A Google Cloud project with **Maps JavaScript API**, **Places API**, and **Distance Matrix API** enabled
- A Razorpay or Stripe account (payments)
- A Resend/SendGrid account (transactional email)
- A Twilio account (SMS)
- A Google Tag Manager / GA4 / Meta Business account (analytics)

## 5. Environment Variables

See `.env.example` in the project root — copy it to `.env.local` and fill in real values before
deploying to production. The app runs in development with all fields blank (integrations are
architecturally stubbed, see README.md).
